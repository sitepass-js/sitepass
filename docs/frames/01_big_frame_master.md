# 1단계 큰틀 기준표

v23.7.392는 실제 기능을 고치기 전에 SitePass 전체를 큰 프로그램 영역으로 나누는 기준판입니다.
이 단계에서는 기존 동작을 바꾸지 않습니다.

## SitePass 큰틀 14개

| 번호 | 큰틀 | 담당 범위 | 섞이면 안 되는 것 |
|---:|---|---|---|
| 1 | 운영/배포 | stable/test/backup, GitHub 업로드, 버전관리 | 기능 수정과 섞지 않기 |
| 2 | 회원/로그인 | 로그인, 로그아웃, 자동로그인, 내정보, 회원탈퇴 | 회원가입 약관/SMS와 섞지 않기 |
| 3 | 회원가입/인증 | 계정 생성, 회원가입 약관, SMS 인증, 아이디/비밀번호 생성 | 기사/인부 서류약관과 섞지 않기 |
| 4 | 약관/개인정보 | 회원가입 약관, 장비/기사/인부 동의, 민감정보 동의 | SMS 문구와 섞지 않기 |
| 5 | 장비·기사·인부 등록 | 등록 폼, 파일첨부, 사진찍기, 만료일, 등록완료 | 보관함 표시 로직과 섞지 않기 |
| 6 | 보관함 | 현재 회원 장비, 관리자 전체 장비, 삭제, 목록 카드 | 회원가입/로그인과 섞지 않기 |
| 7 | QR/공유 | QR, 담당자 링크, 문자공유, 카카오공유 | 인증 SMS와 섞지 않기 |
| 8 | 담당자 수신 화면 | 링크 열람, 서류 확인, 다운로드, 프린트 | 회원 화면/관리자 화면과 섞지 않기 |
| 9 | 알림 | 인증문자, 만료알림, 작성중단, 결제알림, 관리자알림 | 공유문자/인증문자 구분하기 |
| 10 | 결제/요금제 | 무료 베타, 유료전환, 결제연장, 미결제 정지 | 보관함 삭제정책과 섞을 때 주의 |
| 11 | 관리자 | 회원목록, 장비목록, 문자로그, 만료, 결제, 피드백 | 일반회원 보관함과 섞지 않기 |
| 12 | PWA/설치 | 바탕화면 설치, manifest, service worker | 로그인 인증 흐름과 섞지 않기 |
| 13 | Supabase 서버/DB | 회원, 약관, 인증, 장비, 공유, 권한, Edge Functions | 화면 디자인과 섞지 않기 |
| 14 | 디자인 | 공통 CSS, 페이지 CSS, 로고, 버튼, 카드 | 기능 로직과 섞지 않기 |

## 큰틀 수정 원칙

- 큰틀 하나를 고칠 때 다른 큰틀은 건드리지 않는다.
- 예외가 필요하면 수정 전 이유를 기록한다.
- 실사용판에서는 치명 오류만 반영한다.
- 디자인 수정은 기능 수정과 같은 버전에 섞지 않는다.
