# 실사용 배포 보호 규칙

테스트 사용자가 들어오기 시작하면 아래 규칙을 지킵니다.

## 버전 이름
- stable: 실제 사용자가 쓰는 안정판
- test: 새 수정 확인판
- backup: 되돌릴 수 있는 보관판

## 배포 순서
1. 기준판에서 문제 하나 선택
2. test 버전으로 수정
3. 여기 대화창 체크리스트로 확인
4. 문제가 없으면 stable로 반영
5. 이전 stable ZIP 보관

## 한 번에 고치지 말 것
- 회원가입 문자 + 보관함 + 디자인을 한 버전에 같이 고치지 않기
- QR 문제와 관리자 문제를 같이 고치지 않기
- Supabase 권한 문제와 화면 CSS 문제를 같이 고치지 않기

## 실제 수정 전 문장
앞으로 수정할 때는 항상 아래처럼 시작합니다.

이번 수정 범위:
- 무엇을 고칠지

건드리지 않는 범위:
- 무엇을 절대 안 건드릴지

확인할 것:
- 수정 후 어떤 화면을 볼지
