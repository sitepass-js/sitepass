# 실사용판 / 테스트판 / 백업판 운영 기준

## 1. 안정판
사람들이 실제로 쓰는 버전입니다. 치명 오류만 반영합니다.

## 2. 테스트판
수정본을 먼저 올려서 내부 확인하는 버전입니다. 실사용자에게 바로 공유하지 않습니다.

## 3. 백업판
문제가 생기면 즉시 되돌릴 수 있는 ZIP입니다.

## 4. 배포 순서
1. 테스트판에서 수정
2. 내부 확인
3. 핵심 체크리스트 확인
4. 안정판 반영
5. 안정판 반영 후 회원가입/로그인/보관함/QR 재확인
6. 문제 발생 시 즉시 백업판 복구

## 5. 실사용 안정판 반영 가능한 오류
- 회원가입 불가
- SMS 인증 불가
- 로그인 불가
- 보관함 불가
- 장비등록 불가
- QR/담당자 링크 불가
- 관리자 확인 불가

## 6. 실사용 안정판에 바로 반영하지 않는 것
- 디자인 취향 수정
- 새 기능 추가
- 구조 대공사
- 알림 전체 구현
- 결제 기능 추가
