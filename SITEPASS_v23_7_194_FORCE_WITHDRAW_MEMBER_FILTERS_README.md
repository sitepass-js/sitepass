# SitePass v23.7.194 우선 수정 패치

## 이번에 잡는 문제

1. 최고관리자가 회원을 강제탈퇴 처리한 뒤, 같은 브라우저에서 `카카오톡 계정으로 계속하기`를 누르면 기존 Kakao/Supabase 세션 때문에 묻지도 않고 다시 접속되는 문제를 막습니다.
2. 강제탈퇴 기록을 로그인 차단 기준으로 사용합니다.
3. 최고관리자 회원관리에서 회원을 다음 폴더로 나눠 볼 수 있게 합니다.
   - 일반/SitePass
   - 카카오가입자
   - 네이버가입자
   - 강제탈퇴
   - 강제탈퇴-카카오
   - 강제탈퇴-네이버
   - 강제탈퇴-일반

## 적용 방법

원본 최신 HTML을 `/mnt/data/sitepass_v23_7_193_admin_single_role_no_reset.html` 위치에 둔 뒤 실행합니다.

```bash
python /mnt/data/make_sitepass_v23_7_194_force_withdraw_member_filters.py
```

그러면 아래 파일이 생성됩니다.

```text
/mnt/data/sitepass_v23_7_194_force_withdraw_member_filters.html
```

## 확인할 테스트

1. 카카오로 가입/로그인한다.
2. 최고관리자로 로그인한다.
3. 해당 회원을 강제탈퇴한다.
4. 다시 로그인 화면에서 카카오톡 계정으로 계속하기를 누른다.
5. 기존처럼 바로 들어가지 않고, 계정 확인 화면 또는 강제탈퇴 차단 안내가 떠야 한다.
6. 최고관리자 회원관리에서 카카오가입자/네이버가입자/일반/SitePass/강제탈퇴 폴더가 보이는지 확인한다.

## 중요한 운영 메모

이번 패치는 현재 베타 HTML/localStorage 구조에서 바로 증상을 줄이는 수정입니다.
정식 운영에서는 강제탈퇴/차단 회원을 Supabase DB 테이블에 저장하고, 로그인 성공 직후 서버 기준으로 차단해야 다른 휴대폰·다른 브라우저에서도 완전히 막을 수 있습니다.
